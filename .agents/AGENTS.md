# 🛡️ [로컬] 프로젝트 표준 개발 지침서 (Workspace AGENTS Rules)

본 문서는 유튜브 에이전트 프로젝트에서 백그라운드 런봇 서비스 및 API 컨트롤러를 개발/수정할 때 발생할 수 있는 장애를 원천 예방하고, 안전한 실행 환경을 유지하기 위해 모든 코딩 에이전트가 준수해야 할 필수 지침을 정의합니다.

## [로컬] 1. 📡 Flask 백그라운드 서비스 운영 규칙

- **디버그 모드 금지**: 런처 배치 파일(`Runbot_Console_Launcher.bat`)이나 감시 스크립트(`console_guardian.py`)를 통해 백그라운드에서 백엔드 Flask 서버를 띄울 때는 반드시 `debug=False` 옵션으로 실행해야 합니다.
  - *이유*: `debug=True` 시 Flask가 코드 릴로더(Reloader)용 자식 스레드를 추가로 생성하여 백그라운드 프로세스가 이중으로 실행되고 포트 충돌 및 봇 제어 락 교착이 발생할 수 있습니다.
- **CORS 우회 로직 사전 탑재**: 대시보드(Port 8000)와 백엔드 API(Port 5000) 간의 교차 출처 차단을 해제하기 위해, 서버 초기화 시 반드시 preflight(`OPTIONS` 메소드)에 대해 200 OK를 리턴하고 모든 응답에 CORS 허용 헤더(`Access-Control-Allow-Origin: *` 등)를 주입하는 미들웨어를 설치 또는 수동 구현하십시오.

## [로컬] 2. 🤖 비침습적 봇 라이프사이클 관리 규칙

- **봇 코드 보존**: 아우라 및 로파이 등 텔레그램 봇 본체 코드(`telegram_bot/bot.py`)를 임의로 고치는 비효율을 금지합니다.
- **psutil 기반 통제**: 봇의 구동 여부 판단은 `psutil.process_iter`를 통해 메모리 상의 프로세스 인자(`bot.py aura` / `bot.py rofi` 등)를 핀포인트 검출하여 안전하게 판별해야 합니다.
- **실시간 로깅 환경변수**: 봇을 `subprocess.Popen`으로 띄울 때는 버퍼링으로 인한 실시간 출력 지연을 방지하기 위해 `PYTHONUNBUFFERED=1` 환경변수를 강제 전달하고, `stdout`과 `stderr`을 결합한 통합 데몬 스레드로 실시간 에러를 스캔하십시오.

## [로컬] 3. 🧪 통합 연동 자가 검증 의무화 규칙

- **수동 API 자가 검증**: 프론트엔드 HTML 브라우저 자동화 테스트 도구(Playwright 등)가 로컬 환경의 CDP 드라이버 한계 등으로 정상 동작하지 못할 경우, 이를 개발 실패로 단정 짓지 마십시오.
- **검증 시나리오**: 터미널(PowerShell 등)에서 직접 `Invoke-RestMethod` 명령을 사용해 API 게이트웨이의 개별 라우팅 응답성을 직접 교차 증명해야 합니다.
  1. `status` API 조회 결과 OFF 확인.
  2. `control` (START) 호출 성공 확인.
  3. `status` API 재조회 결과 ON 상태 천이 확인.
  4. `control` (STOP) 호출 성공 확인.
  5. `status` API 최종 조회 결과 OFF 상태 및 자원 정리 완료를 교차 증명할 것.

## [로컬] 6. 🎨 고클릭(CTR) 유튜브 썸네일/쇼츠 대시보드 개발 및 모델 매핑 규칙

- **Gemini 이미지 생성 모델 고정**: 구글 AI Studio를 활용한 썸네일 배경 이미지 생성 모듈에서는 반드시 생각하기(Thinking) 메커니즘을 지원하여 프롬프트 추종성이 극대화된 **`models/gemini-3-pro-image` (Nano Banana Pro)** 플래그십 프로 모델을 사용해야 합니다.
  - *이유*: 기존 Imagen 4 모델에 비해 나노 바나나 프로 모델은 빛과 대기 묘사, 소품 구도 배치 등 복잡한 영어 지시어를 훨씬 풍부하고 섬세하게 재현해 냅니다.
  - *호환 endpoint*: 해당 모델은 `generateContent` 메서드를 지원하므로 `v1beta` 경로의 JSON 응답 `inlineData`에서 base64 바이트를 파싱해야 합니다.
- **프롬프트 입력창 상태 락(State Lock) 방어**: Streamlit UI의 📋 최종 이미지 생성 AI 입력용 프롬프트 `text_area`는 고유 세션 Key를 가지므로, 코드 상에서 Value가 업데이트되어도 화면에서 리셋되지 않고 잠겨버리는 현상이 발생합니다.
  - *해결 방안*: 반드시 드롭다운 선택, 토글, 비전 분석 등 입력 장치들의 변화를 감지하는 **Change Detector 로직**을 유지하여 변수 변경 시에만 `st.session_state["active_prompt_text"] = new_prompt`로 세션을 강제 업데이트하고, 사용자가 수동 타이핑하여 편집 중일 때는 상태가 유지되도록 설계하십시오.
- **인물(사람) 제외 필터 강제 적용**: 음악 채널의 웰니스 감성을 저해하거나 자막 시인성을 어지럽히지 않도록, `exclude_people` 체크박스를 도입하여 프롬프트 내의 인물 묘사(`people`, `person`, `crowd`, `silhouettes`)를 자동으로 배제된 풍경 묘사로 가공하고, 프롬프트 미익에 `, empty landscape, no people, no humans` 등 강력한 인물 배제 명령어를 상시 결합해야 합니다.
- **자막 2축(X, Y) 세밀 배치 지원**: 썸네일 자막 오버레이 합성을 위한 Pillow 좌표 변환 시, 메인 타이틀 및 서브 타이틀에 대해 각각 독립적으로 Y축뿐만 아니라 **X축(가로) 조절용 슬라이더**를 제공하여 배경 구도에 따라 자막을 유연하게 피해서 우측/좌측 정렬할 수 있게 해야 합니다.

## [로컬] 10. 📝 Pyrefly 정적 타입 안정성 및 SSAR 보안 통합 관리 규칙

- **Pyrefly 정적 타입 가딩 및 널 참조 방어**:
  - `video_renderer.py` 및 핸들러 코드를 작성할 때는 `from typing import Optional` 및 `from telegram import Message` 등의 어노테이션 누락을 방지합니다.
  - 메시지 처리 전 `if isinstance(raw_msg, Message):` 인스턴스 검사와 `update.effective_chat is None` 조기 반환 가드를 결합하여 `MaybeInaccessibleMessage` 및 `NoneType` 참조 오류를 원천 차단합니다.
  - `Optional[str]` (예: `filter_complex`, `vf`) 타입의 변수를 `list.extend` 등 널 비허용(`Iterable[str]`) 파라미터에 넘기기 전에, `assert var is not None`과 같은 명시적인 타입 가드를 주입하여 린트 경고를 해결합니다.
  - `CallbackUpdateWrapper`와 같이 `Update`를 우회하는 위임 래퍼를 설계할 경우, `Update` 클래스를 상속하도록 명시(즉, `class CallbackUpdateWrapper(Update):`)하여 `bad-argument-type` 형식 불일치를 영구 예방합니다.
  - `query.data`와 같이 `None`일 수 있는 속성을 제어할 때는 `data = query.data or ""` 폴백 처리를 적용하여 `startswith`, `split` 등의 문자열 전용 메소드 널 포인터 참조를 방지합니다.
- **SSAR 자동화 동기화 및 쉘 데드락 방어**:
  - 자동화 동기화 시 로컬 pre-commit 훅에 의한 커밋 거부 버그를 예방하기 위해 `git commit --no-verify`로 커밋을 강제 수행합니다.
  - 비공개 저장소 연동 시 로그인 프롬프트 락(데드락)을 방지하기 위해 터미널에서 수동으로 최초 1회 `git push -u origin master`를 기동하여 윈도우 자격증명 세션을 영구 보존해야 합니다.
  - 보안 감수와 연구 생산성 균형을 위해 `dev_mode: true`를 통한 오버라이드 경고 트랙과 `trusted_hashes` (SHA-256) 바이패스 설정을 조화롭게 운영합니다.

- **자막 2축(X, Y) 세밀 배치 지원**: 썸네일 자막 오버레이 합성을 위한 Pillow 좌표 변환 시, 메인 타이틀 및 서브 타이틀에 대해 각각 독립적으로 Y축뿐만 아니라 **X축(가로) 조절용 슬라이더**를 제공하여 배경 구도에 따라 자막을 유연하게 피해서 우측/좌측 정렬할 수 있게 해야 합니다.

## 10. 📝 Pyrefly 정적 타입 안정성 및 SSAR 보안 통합 관리 규칙

- **Pyrefly 정적 타입 가딩**:
  - `video_renderer.py` 및 핸들러 코드를 작성할 때는 `from typing import Optional` 및 `from telegram import Message` 등의 어노테이션 누락을 방지합니다.
  - 메시지 처리 전 `if isinstance(raw_msg, Message):` 인스턴스 검사와 `update.effective_chat is None` 조기 반환 가드를 결합하여 `MaybeInaccessibleMessage` 및 `NoneType` 참조 오류를 원천 차단합니다.
- **SSAR 자동화 동기화 및 쉘 데드락 방어**:
  - 자동화 동기화 시 로컬 pre-commit 훅에 의한 커밋 거부 버그를 예방하기 위해 `git commit --no-verify`로 커밋을 강제 수행합니다.
  - 비공개 저장소 연동 시 로그인 프롬프트 락(데드락)을 방지하기 위해 터미널에서 수동으로 최초 1회 `git push -u origin master`를 기동하여 윈도우 자격증명 세션을 영구 보존해야 합니다.
  - 보안 감수와 연구 생산성 균형을 위해 `dev_mode: true`를 통한 오버라이드 경고 트랙과 `trusted_hashes` (SHA-256) 바이패스 설정을 조화롭게 운영합니다.
