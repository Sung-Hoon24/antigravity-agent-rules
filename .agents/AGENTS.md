# 🛡️ 프로젝트 표준 개발 지침서 (Workspace AGENTS Rules)

본 문서는 유튜브 에이전트 프로젝트에서 백그라운드 런봇 서비스 및 API 컨트롤러를 개발/수정할 때 발생할 수 있는 장애를 원천 예방하고, 안전한 실행 환경을 유지하기 위해 모든 코딩 에이전트가 준수해야 할 필수 지침을 정의합니다.

---

## 1. 📡 Flask 백그라운드 서비스 운영 규칙

- **디버그 모드 금지**: 런처 배치 파일(`Runbot_Console_Launcher.bat`)이나 감시 스크립트(`console_guardian.py`)를 통해 백그라운드에서 백엔드 Flask 서버를 띄울 때는 반드시 `debug=False` 옵션으로 실행해야 합니다.
  - *이유*: `debug=True` 시 Flask가 코드 릴로더(Reloader)용 자식 스레드를 추가로 생성하여 백그라운드 프로세스가 이중으로 실행되고 포트 충돌 및 봇 제어 락 교착이 발생할 수 있습니다.
- **CORS 우회 로직 사전 탑재**: 대시보드(Port 8000)와 백엔드 API(Port 5000) 간의 교차 출처 차단을 해제하기 위해, 서버 초기화 시 반드시 preflight(`OPTIONS` 메소드)에 대해 200 OK를 리턴하고 모든 응답에 CORS 허용 헤더(`Access-Control-Allow-Origin: *` 등)를 주입하는 미들웨어를 설치 또는 수동 구현하십시오.

---

## 2. 🤖 비침습적 봇 라이프사이클 관리 규칙

- **봇 코드 보존**: 아우라 및 로파이 등 텔레그램 봇 본체 코드(`telegram_bot/bot.py`)를 임의로 고치는 비효율을 금지합니다.
- **psutil 기반 통제**: 봇의 구동 여부 판단은 `psutil.process_iter`를 통해 메모리 상의 프로세스 인자(`bot.py aura` / `bot.py rofi` 등)를 핀포인트 검출하여 안전하게 판별해야 합니다.
- **실시간 로깅 환경변수**: 봇을 `subprocess.Popen`으로 띄울 때는 버퍼링으로 인한 실시간 출력 지연을 방지하기 위해 `PYTHONUNBUFFERED=1` 환경변수를 강제 전달하고, `stdout`과 `stderr`을 결합한 통합 데몬 스레드로 실시간 에러를 스캔하십시오.

---

## 3. 🧪 통합 연동 자가 검증 의무화 규칙

- **수동 API 자가 검증**: 프론트엔드 HTML 브라우저 자동화 테스트 도구(Playwright 등)가 로컬 환경의 CDP 드라이버 한계 등으로 정상 동작하지 못할 경우, 이를 개발 실패로 단정 짓지 마십시오.
- **검증 시나리오**: 터미널(PowerShell 등)에서 직접 `Invoke-RestMethod` 명령을 사용해 API 게이트웨이의 개별 라우팅 응답성을 직접 교차 증명해야 합니다.
  1. `status` API 조회 결과 OFF 확인.
  2. `control` (START) 호출 성공 확인.
  3. `status` API 재조회 결과 ON 상태 천이 확인.
  4. `control` (STOP) 호출 성공 확인.
  5. `status` API 최종 조회 결과 OFF 상태 및 자원 정리 완료를 교차 증명할 것.

## 4. 📂 로컬 에셋 및 설정 경로 조립 규칙

- **하드코딩 절대경로 금지**: 비디오/오디오 파일, 이미지 에셋 경로, VTM 사양서 및 기획/렌더 설정 템플릿 마크다운/JSON 등을 파이썬 코드 내에서 수동 open할 때 특정 개인 로컬 PC 환경에 종속된 절대 경로(예: `c:/1인기업/Apps/유튜브에이전트/` 등)를 선언하는 행위를 엄격히 차단합니다.
  - *이유*: 저장소 이름이 `1.유튜브에이전트` 등으로 미세하게 변경되거나 대표님 로컬 드라이브 기동 환경이 바뀔 때 `FileNotFoundError` 장애가 발생하며, 이는 모듈 임포트 시점부터 시스템 크래시를 유발합니다.
- **실행 환경 기준 동적 조립 필수**: 모든 로컬 사양 파일의 open 경로는 스크립트 실행 디렉토리를 기준으로 하는 `os.path` 조립 방식 적용을 강제합니다.
  - *올바른 코드 예시*:

```python
import os
current_dir = os.path.dirname(os.path.abspath(__file__))
vtm_spec_path = os.path.join(current_dir, "VTM_API_Integration", "vtm_interface_spec.json")
```

## 5. 🛡️ 유튜브 파트너 프로그램(YPP) 수익 창출 자격 유지 및 방어 규정

- **렌더링 동적화 영구 준수**: 비디오 렌더러 엔진(`video_renderer.py`) 수정 시, 프레임 변화량을 보장하는 초미세 색조 순환 변화 필터(`hue='H=0.02*sin(2*PI*t/30)'`) 또는 오디오 반응형 스펙트럼(Showwaves) 등 동적 요소가 비활성화되거나 지워지지 않도록 무조건 유지해야 합니다.
  - *이유*: 시각적 정지 화면 루프가 지속될 경우 유튜브 심사 봇에 의해 '재사용/반복 콘텐츠'로 감지되어 채널 수익화가 비활성화되는 것을 물리적으로 회피하기 위함입니다.
- **오리지널리티 라이선스 명세 강제 결합**: 유튜브 자동 업로드 또는 기획안 설명란(`description`) 빌드 시, 동적으로 `[AURA/GREEN STUDY ORIGINALITY & CREATIVE LICENSE]` 블록을 합성하도록 프롬프트 지침 및 uploader 모듈을 설계해야 합니다.
  - *필수 표준 문구 (영문 전용)*:

```text
This audio content is originally produced and mixed with professional artistic planning dedicated to wellness and emotional stability.
© 2026 [Channel Name]. All rights reserved.
```

- *배치 규칙*: 글로벌 타겟 최적화(English Only 정체성 유지)를 위해 한국어 등의 번역 명세는 일절 노출하지 않으며, 영문 텍스트만 설명란의 가장 하단(Footer) 라이선스 블록 내부에 결합합니다.
- **음향 마스킹 및 영문 글로벌 단일화 의무**:
  - 음향 생성 시 자연음 ASMR의 마스킹 볼륨 비율을 적절히 높여 Content ID의 멜로디 오발령을 차단합니다.
  - Sidebar 채널 정체성을 수립할 때 한국어를 전면 배제하고, 모든 자막/제목/대본을 100% 완벽한 영문(English Only)으로 통일하여 출력하도록 LLM 지침을 엄격히 인코딩해야 합니다.

## 6. 🎨 고클릭(CTR) 유튜브 썸네일/쇼츠 대시보드 개발 및 모델 매핑 규칙

- **Gemini 이미지 생성 모델 고정**: 구글 AI Studio를 활용한 썸네일 배경 이미지 생성 모듈에서는 반드시 생각하기(Thinking) 메커니즘을 지원하여 프롬프트 추종성이 극대화된 **`models/gemini-3-pro-image` (Nano Banana Pro)** 플래그십 프로 모델을 사용해야 합니다.
  - *이유*: 기존 Imagen 4 모델에 비해 나노 바나나 프로 모델은 빛과 대기 묘사, 소품 구도 배치 등 복잡한 영어 지시어를 훨씬 풍부하고 섬세하게 재현해 냅니다.
  - *호환 endpoint*: 해당 모델은 `generateContent` 메서드를 지원하므로 `v1beta` 경로의 JSON 응답 `inlineData`에서 base64 바이트를 파싱해야 합니다.
- **프롬프트 입력창 상태 락(State Lock) 방어**: Streamlit UI의 📋 최종 이미지 생성 AI 입력용 프롬프트 `text_area`는 고유 세션 Key를 가지므로, 코드 상에서 Value가 업데이트되어도 화면에서 리셋되지 않고 잠겨버리는 현상이 발생합니다.
  - *해결 방안*: 반드시 드롭다운 선택, 토글, 비전 분석 등 입력 장치들의 변화를 감지하는 **Change Detector 로직**을 유지하여 변수 변경 시에만 `st.session_state["active_prompt_text"] = new_prompt`로 세션을 강제 업데이트하고, 사용자가 수동 타이핑하여 편집 중일 때는 상태가 유지되도록 설계하십시오.
- **인물(사람) 제외 필터 강제 적용**: 음악 채널의 웰니스 감성을 저해하거나 자막 시인성을 어지럽히지 않도록, `exclude_people` 체크박스를 도입하여 프롬프트 내의 인물 묘사(`people`, `person`, `crowd`, `silhouettes`)를 자동으로 배제된 풍경 묘사로 가공하고, 프롬프트 미익에 `, empty landscape, no people, no humans` 등 강력한 인물 배제 명령어를 상시 결합해야 합니다.
- **자막 2축(X, Y) 세밀 배치 지원**: 썸네일 자막 오버레이 합성을 위한 Pillow 좌표 변환 시, 메인 타이틀 및 서브 타이틀에 대해 각각 독립적으로 Y축뿐만 아니라 **X축(가로) 조절용 슬라이더**를 제공하여 배경 구도에 따라 자막을 유연하게 피해서 우측/좌측 정렬할 수 있게 해야 합니다.

## 10. 📝 Pyrefly 정적 타입 안정성 및 SSAR 보안 통합 관리 규칙

- **Pyrefly 정적 타입 가딩 및 널 참조 방어**:
  - `video_renderer.py` 및 핸들러 코드를 작성할 때는 `from typing import Optional` 및 `from telegram import Message` 등의 어노테이션 누락을 방지합니다.
  - 메시지 처리 전 `if isinstance(raw_msg, Message):` 인스턴스 검사와 `update.effective_chat is None` 조기 반환 가드를 결합하여 `MaybeInaccessibleMessage` 및 `NoneType` 참조 오류를 원천 차단합니다.
  - `Optional[str]` (예: `filter_complex`, `vf`) 타입의 변수를 `list.extend` 등 널 비허용(`Iterable[str]`) 파라미터에 넘기기 전에, `assert var is not None`과 같은 명시적인 타입 가드를 주입하여 린트 경고를 해결합니다.
  - `CallbackUpdateWrapper`와 같이 `Update`를 우회하는 위임 래퍼를 설계할 경우, `Update` 클래스를 상속하도록 명시(즉, `class CallbackUpdateWrapper(Update):`)하여 `bad-argument-type` 형식 불일치를 영구 예방합니다.
  - `query.data`와 같이 `None`일 수 있는 속성을 제어할 때는 `data = query.data or ""` 폴백 처리를 적용하여 `startswith`, `split` 등의 문자열 전용 메소드 널 포인터 참조를 방지합니다.
- **SSAR automation 동기화 및 쉘 데드락 방어**:
  - 자동화 동기화 시 로컬 pre-commit 훅에 의한 커밋 거부 버그를 예방하기 위해 `git commit --no-verify`로 커밋을 강제 수행합니다.
  - 비공개 저장소 연동 시 로그인 프롬프트 락(데드락)을 방지하기 위해 터미널에서 수동으로 최초 1회 `git push -u origin master`를 기동하여 윈도우 자격증명 세션을 영구 보존해야 합니다.
  - 보안 감수와 연구 생산성 균형을 위해 `dev_mode: true`를 통한 오버라이드 경고 트랙과 `trusted_hashes` (SHA-256) 바이패스 설정을 조화롭게 운영합니다.

## [로컬] 23. 🧠 루비아팀 입체적 복합 사고(思考) 및 선제적 방어 진행 규칙 (Preventing AI Tunnel Vision & Passive Verification)

- **23-1. 융합 시나리오 뇌내 시뮬레이션 의무화**:
  - 코드를 수정하기 전에, 비디오 렌더러 엔진이 여러 VFX 플러그인이 연쇄 작동할 때의 타입 변환 흐름과 파일 확장자 간섭을 시뮬레이션해야 합니다.

- **23-2. 선제적 캐시 파기(Invalidation) 프로토콜**:
  - 비디오 생성 공식이나 스킬 연출 파라미터가 수정된 경우, 이전 빌드의 임시 캐시를 선제적으로 삭제·소거하는 클리닝 코드를 반드시 적용해 예전 자산이 재활용되는 버그를 예방하십시오.
- **23-3. E2E 통합 테스트 교차 검증**:
  - 단순 Mocking 테스트에 그치지 말고, 기획안 템플릿 JSON을 해석해 비디오를 직접 빌드하는 E2E 테스트 스크립트를 통해 프레임 전이와 자막 융합이 정상인지 로그와 실물 파일 크기를 끝까지 교차 증명하십시오.

## [로컬] 25. 🛡️ 코드소스 빌드 시 사전 검토 및 방어적 개발 규정 (Lecture Agent Pre-Build Standards)

본 규정은 코드를 전역적으로 빌드하거나 수정할 때 발생할 수 있는 잠재적 크래시 요소를 릴리즈 전에 사전 차단하기 위한 필수 검토 기준입니다.

- **25-1. 3대 Pre-Build 사전 체크리스트 수행 의무**:
  1. **방어적 예외 처리 (try-catch)**: 외부 라이브러리 연동부 및 파일 I/O 구역에 에러를 감지하고 로그를 남길 수 있는 try-catch block이 씌워져 있는가?
  2. **널/빈값 기본값 설정**: 시간, 보고 주기, 채널 키 등의 데이터가 비어있거나 `None`일 때를 대비한 안전한 기본값과 타입 가드 처리가 구현되었는가?
  3. **동적 상대 경로 조립 강제**: 로컬 PC 환경에 종속된 절대 경로를 배제하고, `project_root` 또는 `__file__` 기반 동적 조립 경로를 적용했는가?
- **25-2. 수정 범위 최소화 및 Side Effect 영향도 사전 평가**:
  - 사용자가 요청하지 않은 구조 변경은 일절 금지하며, 오직 기능 추가와 정상화에만 포커스를 맞춤니다.
  - 한 모듈의 수정이 다른 동적 자막, 렌더링 물리 루프 등에 간섭을 일으키지 않는지 영향도를 사전에 평가하십시오.
- **25-3. 텔레그램 API 텍스트 길이 오버플로우 사전 방어**:
  - 외부 프로세스 실패 시 뿜어내는 긴 에러 텍스트를 텔레그램으로 전송할 경우, 문자열 길이가 한계(4,096자)를 초과해 전송 불가 상태가 되지 않도록 3,000자 내외로 슬라이싱 처리해야 합니다.

## [로컬] 26. 🎛️ VFX 튜너 대시보드 락/랙 해결 및 영구 재발 방지 규칙 (Tuner UI/UX Prevention Rules)

- **26-1. HTML 정적 disabled 속성 영구 제거 및 Bootstrap Unlock**:
  - 브라우저의 지독한 임시 캐시나 좀비 DOM이 최초 HTML에 하드코딩된 `disabled` 속성을 붙잡아 layout 모드를 Absolute로 변경해도 슬라이더가 해제되지 않고 잠겨버리는 현상을 방지하기 위해, HTML 마크업 상에서 `disabled` 속성을 하드코딩하는 것을 엄격히 금지합니다.
  - 페이지 로드 즉시 자바스크립트가 강제 잠금 해제(Bootstrap Unlock)를 런타임에서 강제로 주입해야 합니다.
- **26-2. 슬라이더 위치 조작 시 성능 스레드 분리 (`updatePosition` 분리)**:
  - 슬라이더 드래그 시 대량으로 발생하는 `input` 이벤트가 매번 무거운 DOM 재생성(예: `createBars()`)과 CSS 전체 갱신을 유발해 브라우저 메인 스레드가 정지(Lag/Freeze)되는 현상을 방지해야 합니다.
  - X/Y 좌표 위치 이동 시에는 무거운 DOM 재생성을 100% 생략하고, 오직 가상 뷰포트 내의 위치 CSS 스타일(`left`, `top`, `bottom`, `transform`)만 즉각적으로 바꾸어 주는 초경량 갱신 함수(`updatePosition()`)를 매핑하여 조작 랙을 원천 차단하십시오.
- **26-3. Zero-division (분모 0) 산술 가드**:
  - 사용자 슬라이더 조작값 변경에 따른 주파수 너비 분배 시, 나눗셈 인자(`currentBarCount - 1`)는 반드시 논리 OR 폴백(`|| 1`) 처리를 적용하여 분모가 0이 되는 Zero-division 크래시를 전면 격리하고 전체 로직에 `try-catch` 예외 처리를 감싸 안정성을 확보해야 합니다.

## [로컬] 27. 🎬 고클릭(CTR) 유튜브 제목 및 콘텐츠 기획안 프롬프트 표준 템플릿 (High CTR Title Framework)

- **27-1. 후킹 제목 공식 강제 결합 (A도 B도 아니다 룰)**:
  - 기획안 제너레이터(LLM Planner) 및 유튜브 동적 업로드 제목 생성 모듈을 기동할 때, 사람들의 시각적 호기심과 높은 클릭률(CTR)을 유발하기 위해 반드시 아래 **CTR 표준 제목 프레임워크**를 1순위 제목 후보군으로 강제 결합해 출력해야 합니다.
  - **표준 공식**: `[핵심 주제 카테고리] — "[A도 B도 아니다].. [어그로/질문/이것]?"`
  - **대표적 생성 예시**:
    * *자동차 카테고리*: `자동차 — "국산차도 수입차도 아니다.. 요즘 3040이 몰리는 차는?"`
    * *재테크 카테고리*: `재테크 — "적금도 주식도 아니다.. 50대가 노후자금 몰아넣는 곳"`
    * *스포츠 카테고리*: `스포츠 — "손흥민도 김민재도 아니다.. 지금 유럽이 주목하는 한국 선수"`
    * *BGM/웰니스 카테고리 (Aura)*: `Ambient BGM — "백색소음도 ASMR도 아니다.. 요즘 피로한 직장인들이 밤마다 틀어놓는 소리"`
- **27-2. 인지 부조화 템플릿 설계 (The Paradox Hook)**:
  - 스크립트 기획 시, 일반적인 상식(적금, 주식)이나 대명사(손흥민, 김민재)를 1단계에서 전면 부정("~도 아니다")하여 인지 부조화를 일으킨 뒤, 2단계에서 호기심을 유도("~하는 곳/이유")하여 시청 지속성 및 유입률을 극대화해야 합니다.
  - 이 룰은 숏폼(Reels, Shorts) 및 롱폼 기획안 작성 시 LLM Planner의 핵심 후킹 가이드라인으로 탑재하여 작동시켜야 합니다.

## [로컬] 28. 🧚 팅커벨 대시보드 수정 및 배포 무결성 사전 검증 의무화 규정 (Tuner Build Pre-Verification Rules)

- **28-1. 자바스크립트 정규식 내 물리적 개행 예방**:
  - `replace(/\n/g, ...)` 등 정규식 치환 코드를 주입하거나 수정할 때, 개행 문자(`\n`)가 진짜 행 바꿈으로 변해 `Uncaught SyntaxError: Invalid regular expression: missing /` 에러가 터지지 않도록 반드시 한 줄의 인라인 코드로 유지되는지 눈으로 한 번 교차 검토해야 합니다.
- **28-2. 괄호 대칭성 및 세미콜론 종결성 자가 진단**:
  - `tipsMap` 딕셔너리 및 이벤트 리스너를 수정할 때 중괄호(`}`) 또는 소괄호(`)`) 매칭이 어긋나 `SyntaxError: Unexpected token else` 등으로 스크립트 실행이 중단되는 참사가 나지 않도록, `view_file`이나 정밀 자가 컴파일러를 가동해 코드 종결의 무결함을 의무 검토하십시오.
- **28-3. 캐시 파괴(Cache Buster) 및 로컬 파일보안(CORS) 방어**:
  - 모든 이미지 에셋 로딩 주소에 `?t=${Date.now()}` 캐시 버스터가 깨지지 않고 안전하게 매핑되어 있는지 확인하고, 로컬 더블 클릭(`file://`) 대신 `http://localhost:3000` 로컬 웹 서버 주소로만 대표님이 튜닝하도록 권장 접속 안내를 상시 탑재하십시오.
