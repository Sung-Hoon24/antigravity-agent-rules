# Flicker Effect API Specification (P-Based Signal Burst)\n\n## 🎯 기능 목표: '정보의 순간적 과부하' 및 중요 정보 강조.\n\n**원리:** 빛과 어둠이 급격하고 불규칙하게 반복되는 패턴을 통해, 시청자의 주의를 강제로 중앙으로 끌어당기는 효과입니다. (전기적인 스파크나 오래된 CRT 모니터의 노이즈와 유사).\n\n## 📐 수학적 함수 정의:\n\n$$\text{Flicker}(t, P_{\text{rate}}, P_{\text{duration}}) = \begin{cases} Opacity(t) = A \cdot (1 - e^{-k t}) \\ \text{Threshold Condition}: Opacity(t) > T_{min} \land Opacity(t) < T_{max}\end{cases}$$\n\n*   $A$: 최대 밝기 진폭 (Max Amplitude). 노출되는 빛의 최고 강도.\n*   $T_{min}, T_{max}$: 최소/최대 불투명도 임계값. 이 범위 내에서 깜빡임이 발생해야 함.\n*   $P_{\text{rate}}$: 반복 속도 파라미터 (Rate Parameter). 값이 클수록 깜빡임이 빠름.\n*   $P_{\text{duration}}$: 전체 효과의 지속 시간. (짧을수록 충격적).\n\n**💡 구현 가이드:** Flicker는 주로 화면 오버레이(Overlay) 레이어에 적용되어야 하며, 특정 `StateTransition`이 발생했을 때만 호출되어야 합니다. 불규칙성 유지를 위해 $P_{\text{rate}}$와 깜빡임 간의 타이밍은 **랜덤 분포(Random Distribution)**를 따르는 것이 필수적입니다.\n
