# Jitter Effect API Specification (P-Based Vibration)\n\n## 🎯 기능 목표: 시스템의 '미세한 불안정성' 시각화.\n\n**원리:** Sinusoidal Waveform을 기반으로 화면 좌표와 크기(Scale)에 미세하고 무작위적인 떨림을 부여하여, 관찰자에게 '무언가 일어나고 있다'는 잠재적 긴장감을 유도합니다. (학술적 장비의 노이즈/오류 화면 느낌).\n\n## 📐 수학적 함수 정의:\n\n$$\text{Jitter}(t, P_{\text{amp}}, P_{\text{freq}}) = \begin{cases} x(t) = x_0 + P_{\text{amp}} \cdot \sin(\omega t + \phi_{x}) \\ y(t) = y_0 + P_{\text{amp}} \cdot \cos(\omega t + \phi_{y}) \\ s(t) = 1 + P_{\text{amp}} \cdot (1 - e^{-\frac{|t|}{P_{\text{decay}}}})\end{cases}$$\n\n*   $t$: 시간 (Time elapsed in seconds).\n*   $(x_0, y_0)$: 초기 위치.\n*   $P_{\text{amp}}$: 진폭 파라미터 (Amplitude Parameter). 값이 클수록 떨림 폭이 큼. **(핵심 제어 변수)**\n*   $P_{\text{freq}}$: 주파수 파라미터 (Frequency Parameter). 떨림의 속도.\n*   $\phi$: 위상(Phase) 각도. 무작위성을 위해 매번 초기화됨.\n*   $P_{\text{decay}}$: 감쇠율 파라미터 (Decay Rate). 시간이 지남에 따라 진폭이 사라지는 속도 제어. (만약 이 값이 0이면 영구적인 노이즈가 발생함).\n\n**💡 구현 가이드:** Jitter는 배경 레이어 전체 또는 특정 컴포넌트의 변환(Transform) 함수에 적용되어야 하며, 애니메이션 루프에서 지속적으로 재계산되어야 합니다.\n
