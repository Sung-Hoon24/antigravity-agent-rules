# ⚙️ [DHP v2.0] 개발자 핸드오버 패키지 (Technical API Guide)\n\n## ✨ 개요: Codari 통합 테스트용 최종 자산 매뉴얼\n\n본 문서는 유튜브 음악 채널 콘텐츠 제작의 모든 시각적, 기술적 요소를 **코딩 및 구현 가능한 사양(Specification)**으로 변환한 최종 산출물입니다. 이 가이드는 Mockup Kit V2.0과 Asset Bible V2.0을 통합하여 Codari가 End-to-End 테스트용 시뮬레이터를 구축하는 데 필요한 모든 API 레벨의 상세 지침을 제공합니다.\n\n**🎯 목표:** 추상적 디자인 개념 제거 → 수학 함수와 파라미터($P$) 기반의 모듈화된 컴포넌트 구현.\n\n## 📁 디렉토리 구조 안내\n\n- `01_System_Specs/`: 전역 규칙 (Global Constraints) 정의. 색상, 타이포그래피 등 변하지 않는 기준점 제공.\n- `02_Component_Library/`: 재사용 모듈 단위 컴포넌트 라이브러리. 각 컴포넌트는 독립적인 State Machine으로 작동해야 함.\n- `03_Motion_API/`: 동적 시각 효과(Dynamic Visual Effects)의 수학적 정의 및 파라미터 제어 로직.\n\n## 🔑 핵심 작업 지침 (Developer's Checklist)\n\n1. **파라미터($P$) 의존성:** 모든 요소는 하드코딩된 값이 아닌, 명확히 정의된 변수 $P$ 값에 의존해야 합니다. 예: `opacity = sin(time * P_freq) * 0.5 + 0.5`\n2. **State Machine 필수:** 각 컴포넌트와 애니메이션 트랜지션은 반드시 State (예: `IDLE` -> `FOCUS_GAIN` -> `DATA_DISPLAY`)를 거치며 상태 변화에 따른 로직을 가져야 합니다.\n3. **폰트 스케일링:** 텍스트 레이어는 반응형 디자인(Responsive Design) 원칙에 따라, 화면 크기 및 내용의 중요도($P_{importance}$)에 따라 동적으로 크기가 조정되어야 합니다.\n\n**⚠️ 주의:** 모든 에셋은 Deep Indigo & Aged Gold 팔레트를 기반으로 하며, 텍스트 오버레이는 항상 '지적 권위'를 강조하는 미스터리한 분위기를 유지해야 합니다.\n
