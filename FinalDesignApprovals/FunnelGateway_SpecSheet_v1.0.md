# 💠 Funnel Gateway (시스템 결함) 에셋 최종 디자인 사양서 v1.0

**[문서 목적]**
본 문서는 콘텐츠의 핵심 전환 구간(T+320ms ~ T+510ms 예상)에 사용되는 '정보 비대칭성' 시각화 에셋의 기술적, 미학적 표준을 확립합니다. 모든 애니메이터와 개발자는 이 사양을 최우선 기준으로 삼아야 합니다.

**[I. 핵심 디자인 요소 (Visual DNA)]**
*   **콘셉트:** 시스템 오류로 인한 정보 누락/왜곡 (Systemic Deficiency Glitch)
*   **주요 감정:** 충격, 궁금증 증폭, 논리적 결핍 유도
*   **비율:** 16:9 (유튜브 메인 콘텐츠 기준)

**[II. 컬러 팔레트 및 사용 규칙]**
| 이름 | Hex Code | 용도/의미 | 적용 방식 |
| :--- | :--- | :--- | :--- |
| **Deep Indigo (메인)** | `#2D3E50` | 배경, 안정적인 지식 영역. 깊이와 신뢰성 부여. | 가장 많은 비율을 차지하며, 글리치 발생 시 *배경 노출*로 활용. |
| **Aged Gold (강조/정보)** | `#C9B06F` | 핵심 키워드, 중요한 정보 플로우. 권위적 강조. | 텍스트 하이라이트, 데이터 포인트가 정상적으로 전송될 때 사용. |
| **Glitch Magenta (오류색)** | `#E73A8D` | 시스템 오류 발생 시의 왜곡, 아티팩트. 가장 눈에 띄는 색상으로 충격 유도. | 노이즈, 수평/수직 이동(Shifting), 비정상적 색상 반전(Chromatic Aberration)에 사용. |
| **Deep Black (최소화)** | `#0A0A14` | 깊은 그림자 또는 완전한 블랙 아웃 처리. 몰입도를 최대화하며, 정보의 단절을 의미. | 찰나의 순간적 끊김(Cut-off) 효과에만 제한적으로 사용. |

**[III. 애니메이션 및 모션 기술 사양 (Technical Specs)]**
*   **타이밍:** T+320ms부터 시작하여 최대 190ms 동안 지속되는 '최대 밀도' 구간을 목표로 합니다.
*   **핵심 동작: Data Shift & Glitch Cascade**
    1.  **Initiation (T+320ms):** 화면 전체에 `Deep Indigo` 배경 위로 미세한 **노이즈(Noise)** 레이어를 오버레이합니다. (Frequency: Low, Intensity: Medium)
    2.  **Deficiency Trigger (T+350ms):** 핵심 정보 텍스트가 갑자기 불안정해지며 **수평 방향으로 빠르게 흔들립니다.** 이 과정에서 `Glitch Magenta` 색상의 수직선(Scanline effect)이 간헐적으로 지나갑니다.
    3.  **Information Corruption (T+400ms):** 가장 중요한 텍스트 블록의 일부 문자 또는 데이터가 **색상 분리 현상(Chromatic Aberration)**을 일으킵니다. (R/G/B 채널이 미세하게 어긋남) 이로 인해 정보 자체가 신뢰할 수 없다는 느낌을 줍니다.
    4.  **The Void (T+500ms):** 모든 시각 요소가 순간적으로 **블랙 아웃(Deep Black)** 되거나, 매우 빠르게 `Aged Gold`와 `Glitch Magenta`의 노이즈만 남아 강렬한 대비를 이룹니다. 이는 '정보 공백'을 상징하며, 다음 질문/해결책으로 넘어가는 강력한 전환점이 됩니다.

**[IV. 타이포그래피 규칙]**
*   **폰트:** [사용할 폰트명 명시 - 예: Pretendard Bold / Orbitron 등 기술적 느낌의 고딕체]
*   **원칙:** 글리치가 발생해도 핵심 의미는 *잔상(Afterimage)*이나 **부분적인 가독성**으로 남아있어야 합니다. 전체가 지워지는 것보다는, 일부만 깨져서 '아직 정보를 담고 있다'는 긴장감을 유지해야 합니다.
*   **변화:** Glitch가 시작될 때 텍스트의 `letter spacing`이 미세하게 벌어지거나(Stretch), 다음 프레임에서 다시 정상적으로 압축되는 패턴을 적용합니다.
