# 📘 VTM 에셋 라이브러리 API 통합 테스트 매뉴얼 (v3.1)\n\n**[적용 범위]**: 모든 VTM 모듈 (Deep Indigo/Aged Gold 톤)\n**[핵심 원칙]**: 애니메이션의 타이밍(Timing)과 논리적 전환점(Logical Trigger)에 초점을 맞춰 제어합니다.\n\n### I. 필수 기본 파라미터 (Global Parameters)\n| 파라미터명 | 타입 | 설명 | 범위/값 예시 |\n| :---: | :---: | :---: | :---: |\n| `duration` | Float | 애니메이션의 전체 재생 시간 (초). | 0.5 ~ 12.0 |\n| `transition_type` | Enum | 모듈 간 전환 효과 타입. | `[Fade-Out, Slide-Up, Glitch]` |\n| `focus_point` | Float | 시각적 초점 이동 지점 (화면 좌표 비율). | 0.2 ~ 0.8 |\n\n### II. 애니메이션 모듈별 제어 매뉴얼 (Modules)\n\n#### 1. [Module A] 'Concept Reveal' (개념 제시/도입부 활용)\n*   **목적**: 추상적인 개념을 구체화된 비주얼 오브젝트로 순차적으로 공개할 때 사용합니다.\n*   **파라미터 제어**: \n    *   `Reveal_Sequence`: (Int) 몇 단계의 객체가 차례로 나타날지 지정합니다. (예: 3단계)\n    *   `Initial_Opacity`: (Float) 최초 등장 시 투명도. (기본값: 0.1)\n    *   `Scale_Curve`: (String) 크기 변화의 속도 곡선 제어. (`Ease-OutQuad` 권장)\n\n#### 2. [Module B] 'Data Stack & Growth' (데이터 시각화/근거 제시 활용)\n*   **목적**: 복잡한 데이터나 논리 구조가 쌓이는 과정을 역동적으로 보여줍니다.\n*   **파라미터 제어**: \n    *   `Stack_Depth`: (Int) 스택될 객체의 최대 깊이. \n    *   `Connection_Style`: (Enum) 데이터 연결선 스타일. (`[Dashed-Line, Solid-Arc, Glitch-Wire]`) \n    *   `Speed_Factor`: (Float) 정보가 쌓이는 속도 조절. (0.8 ~ 1.5)\n\n#### 3. [Module C] 'The Paradox Shift' (논리적 전환점/VTM Trigger 핵심!)\n*   **목적**: 논지가 급격히 바뀌거나, 시청자가 기존의 믿음(Assumption)에 의문을 갖게 만드는 순간을 연출합니다.\n*   **파라미터 제어**: \n    *   `Shock_Intensity`: (Float) 충격의 강도 (0.0 ~ 1.0). \n    *   `Disruption_Pattern`: (Enum) 파괴 패턴 선택. (`[Glitch-Flicker, Color-Inversion, Noise-Burst]`) \n    *   `Duration_Hold`: (Float) 패러독스 화면을 유지하는 최소 시간(초).\n\n#### 4. [Module D] 'Deep Dive Focus' (심화 논쟁/질문 제기 활용)\n*   **목적**: 특정 개념이나 단어에 시청자의 주의를 강하게 집중시킵니다.\n*   **파라미터 제어**: \n    *   `Zoom_Rate`: (Float) 확대 속도 (초당 픽셀 변화율).\n    *   `Highlight_Cycle`: (Int) 강조되는 단어의 순서 및 개수.\n    *   `Ambient_Effect`: (Enum) 배경에 깔리는 분위기 효과. (`[Subtle-Grain, Deep-Indigo-Halo]`) \n\n#### 5. [Module E] 'Temporal Archive Flow' (시간적 흐름/결론 정리 활용)\n*   **목적**: 시간의 흐름이나 정보의 아카이빙(Archiving) 과정을 상징적으로 보여줍니다.\n*   **파라미터 제어**: \n    *   `Flow_Direction`: (Enum) 흐름 방향 (`[Clockwise, Counter-Clockwise, Random]`) \n    *   `Archival_Density`: (Float) 아카이브된 정보의 밀도. \n    *   `Decay_Rate`: (Float) 시간이 지남에 따라 정보가 희미해지는 속도.\n
