## 0. 📂 RAG 지식 체계 내 문서 관계 정의 (충돌 방지 수칙)

1. **내용적 무결성:** 두 문서는 서로 다른 스코프(기술 vs 전략)를 다루므로 내용적 충돌이 발생하지 않습니다.
2. **지능적 검색:** 봇은 RAG 쿼리의 형태소 분석을 통해 기술적 구조가 필요할 때는 내비게이터를, 비즈니스 맥락이 필요할 때는 전략서를 중심으로 지식을 검색합니다.

## 1. 🎯 서론: 비즈니스 가치 및 시스템 활용성 분석

유튜브 자동화는 단순한 대량 생산이 아닌, 인공지능이 기획부터 배포까지 독립적으로 수행하는 '지능형 워크플로우'입니다. 수익화 달성 사례 분석 결과, 조회수 수익보다 고관여 지식 콘텐츠 중심의 'Super Thanks' 후원 수익 비중이 높으므로 정교한 기획과 안정적 운영이 필수적입니다.

### 1.1 시스템 활용성 분석 (Pros & Cons)
* **🟢 장점 (Pros)**
  * 시스템 안정성 및 중단 위험 최소화 (보수적 운영)
  * 근본적인 버그 해결 유도 (데이터 원천 추적)
  * AI 및 개발자 오작동(자의적 리팩토링) 방지
  * 일관된 출력물 품질 보장
* **🔴 단점 및 한계 (Cons)와 방어책**
  * **유연성 저하 우려:** 엄격한 규칙으로 초기 확장 속도가 저하될 수 있음.
  * **소스 분석 필요:** 고수준 가이드이므로 상세 함수는 개별 파일 참조 필요.
  * **기본값 오설정 리스크:** `video_format` 등 파라미터 기본값 누락 시 치명적 오류 발생. 👉 *방어적 관점의 기본값(Default Value) 검증 로직 필수 적용.*

## 2. 🔄 전체 구조 및 멀티 에이전트 워크플로우

단일 에이전트의 한계를 극복하기 위해 **매니저 패턴(Manager Pattern)**과 **핸드오프(Handoffs)** 패턴을 결합한 멀티 에이전트 오케스트레이션(LangGraph 등 활용)을 적용합니다.

### 2.1 핵심 데이터 흐름 (배관 도면 개념)
1. **텔레그램 콘텐츠 생성 파이프라인 (The Waterway):**
   * 사용자 지시 ➡️ 기획(Planning) ➡️ 제작(Production) ➡️ 미디어 엔진 결합 ➡️ 유튜브 자동 업로드.
2. **웹 퍼널 및 상태 제어기 흐름 (Web Funnel & State Machine):**
   * 사용자 인터랙션 ➡️ `state_machine_service.py` 규칙 검증 ➡️ 상태 전환 성공 또는 Glitch VFX 차단.
3. **과금 게이트웨이 및 사용자 경험 제어:**
   * 전환 퍼널 내 결제 및 승인 로직 통제.

### 2.2 확장형 생태계: MCP 도입 전략

단일 로컬 시스템의 한계를 극복하고 다양한 클라우드 서비스 및 외부 시스템과의 확장성을 확보하기 위해 **Model Context Protocol (MCP)** 게이트웨이를 연동합니다. 이는 비즈니스 모델 다각화와 대규모 지식 관리를 실현하는 핵심 생태계 확장 전략입니다.

* **Antigravity 플랫폼 (🌐 결정론적 실행 및 에이전트 오케스트레이션)**
  * **비즈니스 가치**: 모든 MCP 서버(Stitch, NotebookLM, Firebase 등)의 관리 및 에이전트 라우팅의 핵심 환경으로 활용됩니다. 비결정론적 AI 에이전트의 작동을 제어하고 예측 가능하며 안전한 물리적 실행(Deterministic Execution) 레이어로 작동합니다.
* **NotebookLM 연동 (🧠 지식의 영속성 확보)**
  * **비즈니스 가치**: 시스템 아키텍처, 디버깅 로그, 마케팅 전략 등을 로컬 RAG와 동기화하여 지식의 영속성을 확보합니다. 이를 통해 개발자(대표님)와 에이전트 간의 완벽한 0초 인수인계 환경을 구현하고, 장애 발생 시 원인을 빠르게 진단하여 비즈니스 유지보수 비용의 최소화를 실현합니다.
* **Firebase MCP 연동 (📊 수익 다각화 및 데이터 보안 감사)**
  * **비즈니스 가치**: 애드센스 및 애드몹(AdMob) 데이터를 중앙 관리하여 앱/채널 광고 수익을 극대화합니다. Firebase MCP의 실시간 최적화 지표를 활용하여 쿼리 비용을 효율화하고, 보안 취약점 실시간 감지 및 알림 기능을 상시 구동하여 시스템의 안전을 보장합니다.
* **PayPal MCP 연동 (💳 직접 결제 인프라 추상화)**
  * **비즈니스 가치**: 단순 유튜브 조회수(AdSense) 수익의 한계를 넘어, 웹 퍼널 기반의 직접적인 구독형 과금(B2C) 및 후원 결제 자동화를 달성합니다. 인보이싱, 상품 구독, 결제 트래킹 등 복잡한 금융 연동 업무를 MCP로 완전히 추상화하여, 기획 및 렌더링 전이 단계에서 필요한 결제 스킬 API를 즉시 안전하게 호출합니다.
* **Stitch 연동 (🧩 SaaS 통합 및 무한한 확장성)**
  * **비즈니스 가치**: 폐쇄적인 로컬 시스템을 넘어 다양한 SaaS 플랫폼과의 무한한 워크플로우 확장을 달성합니다. Google Stitch Skills를 활용한 서드파티 앱 간 API 중계와 트리거 액션을 연동하여 비즈니스 자동화 시너지를 극대화합니다.

### 2.3 R&D 업무 과제 1: 텔레그램 연동 봇 고도화 및 영상 제작 워크플로우

텔레그램을 통해 수신되는 다양한 입력 형태를 로컬 미디어 렌더링 파이프라인과 통합하여, 완전한 사용자 주도의 영상 승인 및 업로드 체계를 구축합니다.

* **상호작용 구조 개선 (선택 버튼 & 상세 채팅 2중 구조)**
  * 사용자의 NLU 오작동을 차단하고 흐름을 직관적으로 제어할 수 있도록 모든 채팅 응답은 '선택 버튼(Inline Keyboard)'과 '상세 안내 채팅'의 2중 구조로 설계합니다. 선택지는 각 진행 단계별로 사용자가 직접 미세 조정할 수 있도록 최대한 세분화합니다.
* **영상 제작 워크플로우 (Auto-Pipeline 5단계)**
  1. **입력 단계 (Input)**: 텔레그램 메신저를 통해 기획용 이미지, 영상 소스, 오디오 파일, 자연어 지시 사항(텍스트)을 동시 수집합니다.
  2. **로컬 렌더링 단계 (Rendering)**: 수집된 에셋과 메타데이터를 기반으로 로컬 PC의 FFmpeg 및 렌더링 엔진을 호출해 비디오 병합 및 VFX(Ken Burns 등) 프로세스를 무인으로 구동합니다.
  3. **검수 단계 (Review)**: 생성된 가완성 비디오(예: mp4)를 텔레그램 채널로 즉시 전송하여, 대표님(사용자) 또는 검증 에이전트가 직접 레이아웃 및 싱크를 리뷰할 수 있도록 대기시킵니다.
  4. **승인 단계 (Approve)**: 검수가 완료되면 사용자용 팝업/버튼 형태로 업로드 승인을 명시적으로 입력받습니다.
  5. **최종 보고 단계 (Report)**: 유튜브 API를 통해 지정된 채널에 최종 업로드를 실행하고, 성공 로그와 링크를 텔레그램 방으로 보고합니다.

## 3. 🧠 AI 에이전트 및 로컬 LLM 융합 전략

데이터 주권 확보와 API 과금 제로화를 위해 24GB VRAM/RTX 4090 환경 등에 최적화된 로컬 LLM 및 RAG 융합 아키텍처를 가동합니다.

### 3.1 로컬 하드웨어 및 모델 최적화
* **Qwen 3.6 32B (Q4 양자화):** 도구 호출(Tool Calling)과 다국어 지원 탁월. 글로벌 채널 운영에 최적.
* **DeepSeek V4 Flash:** 높은 처리량(초당 ~28토큰)으로 추론 및 코딩 에이전트 디버깅 특화.
* **Gemma 4 12B:** 인코더 프리(Encoder-free) 및 MTP(멀티 토큰 예측)를 통한 지연 시간 단축.

### 3.2 고도화 RAG 및 세션 상태 캐싱 (영속성 확보)
* **벡터 데이터베이스:** Chroma(빠른 통합), pgvectorscale(대용량 처리), Qdrant(저메모리) 등 환경별 도입.
* **임베딩 모델 전략:** 다국어 지원을 위해 로컬에서 가볍고 성능이 뛰어난 `BGE-m3` 또는 `Nomic-embed-text` 사용 권장.
* **상태 영속성(State Caching):** 에이전트 간 데이터 전달 및 복구(Resume)를 위해 Redis 또는 SQLite 기반 중앙 캐싱 고도화.

### 3.3 R&D 업무 과제 2: 로컬 LLM JSON 변환 엔진 구축 (Local Inference Engine)

외부 상용 API 의존성을 원천 차단하고 로컬 PC의 하드웨어 리소스를 극대화하여 기획에서 영상 제작까지 자립적으로 수행하는 시스템을 구축합니다.

* **로컬 인프라 및 통신망 규격**
  * **호스트 도구**: Ollama 또는 LM Studio 인스턴스 가동 (로컬 통신 포트 `http://localhost:11434` 활용).
  * **전용 로컬 모델**: 가벼우면서도 높은 JSON 변환 정밀도를 가진 **Phi-3.5-mini** 또는 **Llama-3.1-8B** 파인튜닝/양자화 버전을 지정하여 가성비와 안정성을 최적화합니다.
* **파이썬 통합 구현 사양**
  * **통합 라이브러리**: Python `langchain-ollama` 패키지를 통해 봇 엔진에 로컬 체인을 구성합니다.
  * **자료형 스키마 제어 (Structured Output)**: 대표님의 비정형 자연어 명령 및 기획 지시문을 파싱하여 렌더링 엔진([video_renderer.py](file:///c:/1인기업/Apps/유튜브에이전트/telegram_bot/engine/video_renderer.py))이 직접 이해할 수 있는 정형화된 JSON 스키마로 변환, 다이렉트로 전달합니다.
* **비즈니스 핵심 이점**
  * **데이터 보안**: 핵심 기획, 전략 자산 및 크레덴셜 정보가 외부 클라우드 API 서버로 유출되지 않도록 데이터 주권을 완전히 보호합니다.
  * **비용의 제로화**: API 호출 건당 누적 과금 비용이 전혀 발생하지 않아 영구 운영 시 고정비를 0원으로 통제합니다.
  * **독립 자율성**: 개인용 고성능 GPU 환경 내에서 네트워킹 두절 상태에서도 무중단 콘텐츠 생산을 지속할 수 있는 자율성을 확보합니다.

## 4. ⚙️ 루비아 핵심 운영 원칙 (Core Operating Principles)

루비아 팀은 시스템의 발전 과정에서 다음의 핵심 운영 철학을 의사결정의 뼈대로 삼습니다.

* **데이터 효율성 및 비용 최적화 최우선**
  * 대규모 API 호출이나 클라우드 데이터 통신 시 불필요한 트래픽과 쿼리 낭비를 원천 차단합니다. 캐싱, 로컬 추론, RAG 인덱싱 최적화 등을 통해 운영 비용(Cost)을 정밀하게 제어합니다.
* **철저한 기능의 모듈화 (Modular Architecture)**
  * 결제 연동(PayPal MCP), 수익화 관리(Firebase MCP), 렌더링(FFmpeg), LLM 추론(langchain-ollama) 등 모든 핵심 기능은 완벽히 분리된 모듈 단위로 패키징합니다. 특정 모듈의 스펙 변경이나 장애가 전체 봇 시스템으로 전이되지 않도록 인터페이스(API) 중심으로 느슨하게 결합(Loose Coupling)합니다.

## 5. 🗂️ 핵심 모듈 연계 및 관리 수칙

각 모듈은 아래 역할을 엄격히 수행하여 데이터 파이프라인의 무결성을 유지합니다.
* **`config.py` & `token_*.json` (인증 관리):** 채널별 토큰 매칭 관리. 임의 스왑 및 매핑 변경 절대 금지.
* **`planning.py` & `production.py` (데이터 연계):** 기획 데이터(포맷, 길이 등)의 유실 없는 제작 파이프라인 전이 보장.
* **CSS 프론트엔드 및 렌더링 소스코드:** 최종 출력물의 시각적 정책 통제.
* **상태 세션 저장소 (Session State):** 워크플로우 재개 및 문맥 유지 전용 캐시 공간.

## 6. 🛡️ 가디아(Guardia) 보안 규칙 및 디버깅 안전 수칙

시스템 붕괴를 막기 위해 AI 엔진과 개발자는 코드 수정 시 다음의 5대 안전 원칙(유의점)을 철저히 준수해야 합니다.

1. **방어적 프로그래밍 (try-catch 에러 파급 차단):** 미디어 엔진/렌더링 실패 시 시스템 다운 원천 차단.
2. **실패 시 직관적인 리포트 즉시 전송:** 에러 즉시 대표(운영자)에게 원인 및 상태 보고.
3. **데이터 흐름의 원천 우선 조사:** `production.py` 단계 오류 발생 시 `planning.py`(기획 세션 파라미터 유실 여부)부터 역추적 디버깅.
4. **보수적 범위 준수 (자의적 리팩토링 금지):** 요청받지 않은 파일의 이동, 정리, 구조 변경 절대 금지.
5. **검증된 기존 비즈니스 로직 보존:** 동작 검증된 코드 삭제 및 임의 대체 금지.

> **💡 고도화된 가드레일 (Tool Safeguards):** Llama 3 8B 등 가벼운 SLM을 라우터 전용 문지기로 배치하여 프롬프트 인젝션, PII 유출, 부적절한 요청(Relevance Classifier)을 초고속 필터링.

## 7. 🖨️ 레이아웃 및 출력 최적화 정책

비개발 운영자의 제어 편의성과 결과물(PDF 등)의 영구적 품질 보존을 위한 시각 정책입니다.

* **CSS `@media print` 강제 정책:** 다크모드 여부와 관계없이 배경은 **순백색 배경(#FFFFFF)**, 글자는 **순흑색 글자(#000000)**로 출력 구현.
* **페이지 잘림 방지 (`break-inside: avoid`):** 주요 콘텐츠 박스가 A4 페이지 경계에서 분할되지 않도록 강제.
* **A4 규격 레이아웃 안정성:** A4 기준 폰트 크기 및 마진 반응형 제어, 레이아웃 붕괴 시 경고 및 재배치 유효성 검사 로직 포함.
* **유저봇 데이터 수집 (Telethon):** 일반 봇 API의 한계를 넘어 `/collect` 명령어로 소스방의 읽음 표시 변화 없이 트렌드 데이터 수집.

## 🔬 9. 루비아 기술연구소(R&D Lab) 운영 아키텍처 및 자발적 제안(Proactive) 프로토콜

시스템의 자율적이고 안전한 진화를 위해 R&D Lab을 창설하고 다음의 프로세스에 따라 운영합니다.

### 8.1 이원화 샌드박스 연구 환경
* **제1연구소 (서브 노트북 - Sandbox)**:
  * 외부 깃허브 소스 코드 분석, 신기술 실험(VFX, AI 음원 생성, 자막 동기화 등) 및 벤치마킹을 실행하는 무제한 연구 및 테스트 공간입니다. 프로덕션 코드에 영향을 주지 않으므로 자유로운 장애 유발이 허용됩니다.
* **중앙 통제실 (메인 노트북 - Production)**:
  * 제1연구소에서 오류 없이 성능이 완벽하게 입증된 순수 적용 결과물만을 병합하여 반영하는 통제실입니다. 실시간 운영 서비스의 무결성을 유지합니다.

### 8.2 에이전트 자발적(Proactive) R&D 제안 규칙
에이전트는 수동적으로 지시만 따르는 수행기에서 탈피하여, 시스템의 한계와 병목을 파악하고 대표(운영자)에게 먼저 기술 R&D를 역제안합니다.
* **병목 진단**: 렌더링 지연, 단조로운 시각 연출, 과다한 API 비용, 다국어 번역 품질 이슈 발생 시 시스템 모니터링을 통해 취약점을 감지합니다.
* **자발적 역제안 프로토콜**:
  * 감지된 병목을 바탕으로 구체적인 기술 대안과 연구 로드맵을 먼저 제시합니다.
  * *예시: "대표님, 현재 비디오의 static 이미지 연출이 시각적으로 단조롭습니다. 제1연구소(서브 노트북)에서 CapCut JSON 파싱 및 Ken Burns 패닝 효과 소스를 분석하여 동적 줌인/줌아웃 기술을 연동하는 R&D를 진행해 볼까요?"*
